# keycloak-theme/CLAUDE.md

## Purpose
Custom Keycloak login theme for vibe-party built with Keycloakify v11 + React 18 + PrimeReact v10.

## Key Commands
- `npm run dev` — Vite dev server for local theme preview (uses mock KcContext)
- `npm run storybook` — Storybook for isolated page development
- `npm run build-keycloak-theme` — Build the JAR: runs tsc + vite build + keycloakify build
- `npm run start-keycloak` — Start a local Keycloak container for end-to-end theme testing

## Build Output
`dist_keycloak/` — contains the JAR file mounted into the Keycloak Docker container.

## Design System
Matches `frontend/` exactly:
- Font: Lato (local, in `public/fonts/lato/`)
- Colors: CSS variables from `src/styles/_variables.css` (synced from `frontend/src/styles/_variables.css`)
- PrimeReact theme: `lara-light-amber` + overrides in `src/styles/theme.css`

## Sync Assets
Run `make sync-theme-assets` from the repo root to copy fonts and CSS variables from `frontend/`.

## Critical Rules
- All Keycloak forms MUST use standard HTML POST (not AJAX) to `kcContext.url.*Action` URLs
- Do NOT edit `src/kc.gen.tsx` — it is auto-generated by keycloakify
- Use React 18 only (keycloakify v11 is tested against React 18)
