# keycloak-theme/CLAUDE.md

## Purpose
Custom Keycloak login theme for vibe-party built with Keycloakify v11 + React 18.
Uses `@radix-ui/themes` v3 for UI components — same library as the main frontend.

## Key Commands
- `npm run dev` — Vite dev server for local theme preview (uses mock KcContext)
- `npm run build-keycloak-theme` — Build the JAR: runs tsc + vite build + keycloakify build
- `npm run start-keycloak` — Start a local Keycloak container for end-to-end theme testing

## Build Output
`dist_keycloak/` — contains two JAR files mounted into the Keycloak Docker container.
Docker Compose mounts: `./keycloak-theme/dist_keycloak:/opt/keycloak/providers:ro`

## Design System
Matches `frontend/` exactly — same Radix UI theme config:
- `<Theme accentColor="orange" grayColor="slate" radius="medium">` in `VibePartyProvider.tsx`
- `--accent-9: #ff6b35` override in `theme.css` (Atomic Tangerine)
- Font: Lato (local woff2 files at `src/styles/fonts/*.woff2` — bundled by Vite)
- Brand tokens: `src/styles/_variables.css` (standalone — do NOT overwrite from frontend)

## Component Architecture
```
src/components/
  VibePartyProvider.tsx  — Radix <Theme> wrapper + CSS imports
  AuthCard.tsx           — full-page card layout with floating emoji bubbles
  Logo.tsx               — "vibe" + "party" wordmark
```

## CSS Import Order (VibePartyProvider.tsx)
```
@radix-ui/themes/styles.css   ← Radix base
../styles/theme.css           ← fonts + _variables + accent override
```

## Radix UI Usage in Pages
- `Button` — `type="submit"` works natively; `loading` prop shows spinner
- `TextField.Root` — renders `<input>` with `name` forwarded; put eye-toggle in `<TextField.Slot side="right">`
- `Callout.Root` / `Callout.Text` — error/success messages
- `Flex`, `Grid`, `Text`, `Code` — layout and typography
- `EnvelopeClosedIcon`, `EyeOpenIcon`, `EyeClosedIcon`, `ExclamationTriangleIcon` from `@radix-ui/react-icons`

## Important: "Remember Me" Switch
`Switch` from `@radix-ui/themes` is used for "Remember Me". It does not submit a `name` value
in HTML POST, so it is paired with a hidden input that mirrors the checked state:

```tsx
const [rememberMe, setRememberMe] = useState(login.rememberMe === "on");

<Switch id="rememberMe" checked={rememberMe} onCheckedChange={setRememberMe} size="1" />
<input type="hidden" name="rememberMe" value={rememberMe ? "on" : ""} />
<label htmlFor="rememberMe"><Text size="2">{i18n.msgStr("rememberMe")}</Text></label>
```

Keycloak reads `rememberMe=on` from the raw POST body — the hidden input carries this value.

## Critical Rules
- All Keycloak forms MUST use standard HTML POST (not AJAX) to `kcContext.url.*Action` URLs
- Do NOT edit `src/kc.gen.tsx` — it is auto-generated by keycloakify
- Use React 18 only (keycloakify v11 is tested against React 18)
- Font paths in `_fonts.css` use relative `./fonts/...` URLs — Vite rewrites these for the JAR
- `_variables.css` is standalone — do NOT sync/overwrite from `frontend/src/styles/theme.css`
  (that file is Radix-specific and won't work here as-is)

## Sync Assets
Run `make sync-theme-assets` from repo root to copy Lato font woff2 files from `frontend/`.
This updates both `public/fonts/lato/` (dev server) and `src/styles/fonts/` (Vite bundle).
